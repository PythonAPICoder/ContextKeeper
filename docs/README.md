# ContextKeeper Documentation

ContextKeeper is an intelligent Ollama-compatible middleware layer. Instead of connecting AI clients directly to Ollama, clients connect to ContextKeeper.

Primary planning documents:

- [Project History](PROJECT_HISTORY.md) records completed phases, current state, and approved planning changes.
- [Roadmap](ROADMAP.md) records the active Version 1 release sequence and future-version roadmap.
- [ContextKeeper Future Ideas](FUTURE_IDEAS.md) preserves v2+ and later ideas that are not Version 1.0 commitments.

Version 1 remains focused on a production-quality release. Future ideas such as multi-user architecture, workspace memory, routing, plugins, multi-server orchestration, voice, agents, and advanced analytics should be captured for later versions instead of expanding v1 unnecessarily.
